AI Usage Disclosure Requirement

Any significant use of AI tools (code generation, content creation, etc.) must be clearly disclosed in your project README documentation. Failure to disclose AI usage that we determine to be significant will result in point deductions based on the extent of undisclosed usage.

üö® MUST READ - Demo Requirements

A clear and thorough demonstration is critical for assessment. All team members must be prepared to:

  • Provide detailed code explanation: Walk through your codebase, explaining the role and functionality of each key file and how they integrate in your project folder structure (VS-Code).
  • Execute live, functional demo: Demonstrate the application running successfully. You must explicitly show database integration in action.
  • Explain puzzle mechanics: Clearly articulate how your puzzle implementation works, including move validation, win conditions, and any special features.
  • Demonstrate theme implementation: Show how your holiday theme is consistently applied throughout the application.
  • Explain your architectural decisions: Be prepared to explain your architectural decisions, database design, and any challenges you encountered.

Note: All team members must participate in the demo and be able to explain any part of the codebase.

TA Grading Instructions

  • Use the verification checklist before starting detailed grading
  • Test all features systematically - verify puzzle mechanics, database functionality, and custom features
  • Check puzzle mechanics thoroughly - validate move restrictions, win conditions, and multiple puzzle sizes
  • Evaluate theme implementation - ensure consistent holiday theme throughout application
  • Check source code for proper JavaScript implementation and database integration
  • Verify GitHub repository contains complete project with meaningful commit history
  • Check README for AI disclosure and apply penalties for undisclosed usage (15-30 points based on extent)
  • Evaluate team collaboration through commit history and project documentation
  • Document specific issues when deducting points for future reference
  • Focus on technical implementation - this is now worth 45% of the total grade
  • Pay special attention to demo requirements - ensure students can explain their code and demonstrate functionality
