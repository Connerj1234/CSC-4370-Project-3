This document outlines the grading criteria and requirements for your Christmas Fifteen Puzzle Project. Follow these guidelines to ensure your project meets all expectations.

Part 1: Technical Requirements

Your Christmas Fifteen Puzzle must include the following core technical features:

  • Three Puzzle Versions: Implement all three themed versions (Santa's Workshop, Reindeer Games, Elf AI Workshop) with unique mechanics for each.
  • Database Integration: Use MySQL with proper schema design, security measures, and performance optimization.
  • User Authentication: A secure system for users to register, log in, and manage profiles.
  • Responsive Design: Ensure the application works seamlessly across different devices and screen sizes.
  • Game Analytics: Track user sessions, puzzle completion times, moves, and performance metrics.

Part 2: Team Process & Methodology

All team members will share responsibility for the following roles throughout the project lifecycle.

üéÑ Team Roles & Responsibilities

DESIGNERS (Everyone)

  • Collaboratively define final project requirements and specifications
  • Create user interface mockups and wireframes for all three puzzle versions
  • Design the application architecture and database schema
  • Write pseudocode for complex algorithms and game mechanics

PROGRAMMERS (Everyone)

  • Collaboratively implement the frontend and backend code
  • Ensure code quality, consistency, and proper documentation
  • Implement security measures and performance optimizations
  • Integrate all components into a cohesive application

TESTERS (Everyone)

  • Develop comprehensive test plans with specific procedures
  • Track, report, and prioritize bugs using issue tracking
  • Perform cross-browser and cross-device compatibility testing
  • Gather feedback for final presentation and documentation

üìã Required Agile Methodology

You are required to use Kanban or Scrum framework to manage your project development.

  • Regular Stand-ups: Your team must meet at least every two days to sync progress
  • Sprint Planning: In each meeting, define specific tasks for the next development cycle
  • Documentation: Maintain meeting logs and progress tracking for your final presentation

Presentation Requirement:

Your final presentation must include a dedicated section explaining:

  • How your team implemented Scrum/Kanban methodology
  • Specific benefits this approach provided for your project
  • Challenges faced and how you overcame them

Part 3: Final Deliverables

üé§ The Presentation (20-25 minutes)

Your team presentation must include the following components:

  • Project Introduction: Explain the Christmas Fifteen Puzzle concept and the problem you're solving
  • Design Overview:
    • Show your UI/UX designs and mockups
    • Explain your application architecture and database design
    • Demonstrate key algorithms and game mechanics
    • Highlight the technical implementation of your chosen version
  • Development Process: Explain your team's workflow, challenges faced, and solutions implemented
  • Testing Methodology: Describe your testing approach, including test procedures, data, and bug tracking
  • Live Demonstration: Showcase a working version of your application with all key features
  • Code Repository: Present your GitHub repository with proper documentation

Teamwork Requirements:

All team members must contribute meaningfully to the presentation. You can alternate speakers or have dedicated presenters for different sections.
